## v1.4.10

### 更新亮点

- 修复 Windows 桌面端首次安装或冷启动时，前端偶发 `connect ECONNREFUSED 127.0.0.1:18765` 的核心服务连接报错。
- Electron 主进程所有 FastAPI 调用统一走后端 ready / retry 控制；如果 `/health` 之后请求阶段短暂断连，会重新确认并尝试恢复后端。
- 后端彻底不可用时，错误信息会带上桌面日志路径、Python 路径和资源目录，方便用户把真实启动原因发回排查。
- 桌面打包阶段如果缺少内置 Python bundle 会直接失败，避免产出安装后必然无法启动核心服务的 Windows 包。
- 新增 Node 回归测试，覆盖请求期连接拒绝恢复、失败诊断包装和 Python bundle 缺失时的打包校验。
